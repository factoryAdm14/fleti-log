# ROADMAP Fleti 2026–2028

Planejamento estratégico do ecossistema **Fleti Enterprise** após a conclusão do Master Plan v4.0 (FASE 000–019).

**Versão:** 1.0  
**Branch base:** `feature/fleti-enterprise-v4`  
**Produção:** https://fleti.com.br  
**Data:** 2026-06-26

---

## 1. Resumo executivo

O Fleti v4.0 consolidou auditoria, correções críticas, modernização visual, PIX (Mercado Pago + EFI), multi-stop delivery (API), segurança, testes, deploy e observabilidade. O roadmap 2026–2028 foca em **produto**, **escala**, **inteligência operacional** e **confiabilidade** — sempre com feature flags e compatibilidade retroativa.

### Princípios (herdados do Master Plan)

1. Nunca remover funcionalidades existentes sem migração explícita.
2. Toda feature nova via **feature flag** (desligada por padrão).
3. Apenas **novas migrations** — sem alterar tabelas legadas destrutivamente.
4. APIs retrocompatíveis; versão mínima de app controlada no admin.
5. Uma entrega por fase; relatório técnico ao final.

## 8. Stack técnica evolutiva

| Camada | Hoje (2026) | 2027 | 2028 |
|--------|-------------|------|------|
| Backend | Laravel 12 monólito modular | + Redis queue, OTel | Serviços PIX/notify |
| DB | MySQL Hostinger | + índices, read replica? | Sharding por cidade (avaliar) |
| Cache | File/DB | Redis | Redis cluster |
| Logs | `fleti-ops` + laravel.log | Elasticsearch | Centralizado + alertas |
| Apps | Flutter 3.2 User/Driver | + OCR/ML on-device | Modular flavors |
| Infra | FTP + SSH deploy | Docker staging | K8s ou managed (avaliar) |

## 9. Feature flags — convenção

| Padrão | Exemplo |
|--------|---------|
| Admin toggle | `enable_multi_stop_delivery` em `parcel_settings` |
| Gateway | `is_active` em `settings` (payment) |
| App config API | `enable_*` no `/api/customer/configuration` |
| Env only | `FLETI_FEATURE_OCR=false` para dev |

Toda flag nova: **default OFF**, documentação em `*_GUIDE.md`, rollback = desligar toggle.

## 11. Riscos e mitigação

| Risco | Mitigação |
|-------|-----------|
| Regressão em wallet/corridas | Testes + feature flags + rollback |
| LGPD (facial, telemetria) | Consentimento explícito, DPO, retenção limitada |
| Custo APIs (Maps, WhatsApp, ML) | Quotas, cache, fallbacks |
| Hostinger limits | Monitorar CPU/RAM; plano upgrade ou migração 2027 |
| Dívida técnica monólito | Extrair PIX/notify primeiro (menor acoplamento) |
